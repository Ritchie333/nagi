Description

NAGI (New Adventure Game Interpreter) is a clone of Sierra's own AGI which they created and used through the 80's to produce a whole bunch of great adventure games like Space Quest, Leisure Suit Larry and Kings Quest. NAGI was created by disassembling the original AGI executable and writing equivalent C code that would run under SDL which is a free generic library for low-level access to graphics and audio. It can read both v2 and v3 game data for the PC.

I started working on this (disassembling and learning stuff) around November 1999. I didn't really do much work on it until the start of 2001 when I finished the picture rendering code. So I started with a program that draws agi pics to this... I actually got the initial idea from the guy who recreated an old game called Digger . He did the same thing by disassembling the original game and recoding it all in C.

The original name for NAGI was GI-Jay.. like GI-Joe only different.  It evolved from JayGI.  NAGI sounds a bit more sophisticated anyways and can be confused with a whole bunch of things !

Features

  • Full Windows program running under SDL
  • Accurate compatibility with original AGI games.
  • Supports both version 2 and version 3 type data.
  • Mouse Support!  (Buffbri and original Sierra implementation like Amiga and Mac)
  • Three channel music. (using sampled sound)
  • Uses the SDL library so should be easy to port to other systems
  • CRC32 id system to determine which game you're playing.
  • Easy to change the game id.
  • No stack patches/errors .

Instructions

  • Download the NAGI executable and the SDL library from the download section
  • Find an AGI game.
  • Get the required files for the AGI game to work:
    • Version 2 games:
      • words.tok
      • object
      • snddir
      • picdir
      • viewdir
      • logdir
      • vol.0
      • vol.1
      • vol.etc.etc.etc
    • Version 3 games where <gameid> is replaced by the game's actual id (ie KQ4 for Kings Quest 4):
      • words.tok
      • object
      • <gameid>dir
      • <gameid>vol.0
      • <gameid>vol.etc.etc.etc
  • Copy NAGI, SDL and the game files into one directory
  • Run n.exe

Download

The latest version is v2.03.  In order to run NAGI you need the NAGI executable and the SDL dynamic linking library .  SDL used to be packaged with NAGI but this was dodgy and eventually somebody was going to complain.  Just copy them both in the same directory and you should be right.

NAGI v2.03 - selectable games, dir search, cga, responsive input, amiga support!
SDL DLL v1.2.1 - A great library that allows portable access to the system's video and sound libraries.  Check out their site for more information and latest versions.

Bugs

  • Parsing errors have occurred but I think they've been fixed.
  • In some games that use an older interpreter (like Space Trek), sprites disappear.  This is consistent with the later Sierra interpreters and Sarien however.
  • Sound with soft clicks at end of notes.
  • Computer slows down if NAGI crashes so SDL isn't shutdown properly. Eventually, you will have to reset if it crashes too many times because the computer will be too bogged down.
  • Sound sounds horrible if you don't set the SDL driver to use waveout. (in nagi.ini)
